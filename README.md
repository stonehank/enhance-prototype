
注入prototype：所有原生创建的实例原型链都会受到改变

非注入prototype：只有通过enhance创建的才有新增，原生创建不受影响，可以通过toRaw返回原始值

控制一体化：多了一层prototype，但方便操作，任何实例都可以对class管理

控制分离：实例不可以对class进行管理，需要用额外的管理器，但方法共享




|        |注入prototype|非注入prototype|控制一体化|控制分离|
|--------|:-----------:|:-------------:|:--------:|:------:|
|基本类型|     ✔      |               |          |    ✔   |
|引用类型|             |       ✔      |          |    ✔   |
